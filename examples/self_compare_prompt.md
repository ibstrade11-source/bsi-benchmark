# دستورالعمل خودمقایسه‌گری برای تحلیلگر (Self-Comparison Prompt)

این فایل برای زمانی است که یک تحلیلگر (معمولاً یک مدل زبانی) دو تحلیل از
یک مقاله نوشته — یکی «raw» (استاندارد) و یکی «bsi» (طبق چارچوب شاخص
بهمنش) — و حالا باید خودش این دو تحلیل را مقایسه کند.

## اصل طراحی (چرا این فرمت این‌شکلی است)

این ابزار **هیچ معیار یا روبریک ثابتی به شما تحمیل نمی‌کند**. شما (تحلیلگر)
باید خودتان تصمیم بگیرید چه معیارهایی برای قضاوت کیفیت این دو تحلیل مهم
است. دلیلش: اگر معیار از بیرون تحمیل شود، دیگر این یک خودارزیابی مستقل
نیست. تنها چیزی که این ابزار مشخص می‌کند **شکل ثبت** است، نه محتوای
معیارها.

## کاری که باید انجام دهید

۱. هر دو تحلیل خودتان (raw و bsi) را که قبلاً نوشته‌اید، جلوی خودتان
   بگذارید. شما می‌دانید کدام کدام است — این طبیعی و بخشی از ارزش
   خودارزیابی است.

۲. فهرستی از معیارهایی که خودتان برای قضاوت کیفیت این دو تحلیل مهم
   می‌دانید بنویسید (هرچند تا که لازم می‌دانید؛ می‌تواند شناسایی علّی،
   عمق مکانیسمی، صداقت در بیان محدودیت‌ها، یا هرچیز دیگری باشد که شما
   مرتبط می‌دانید).

۳. برای هر معیار، به هر دو تحلیل یک نمره روی مقیاسی که خودتان انتخاب
   می‌کنید (مثلاً ۱ تا ۱۰) بدهید. برنده هر ردیف خودکار از روی این دو
   عدد محاسبه می‌شود — شما لازم نیست جداگانه «برنده» را اعلام کنید.

۴. یک قضاوت کلی نهایی بدهید (raw / bsi / برابر) و دلیلش را به زبان خودتان
   توضیح دهید.

## فرمت خروجی لازم (JSON)

خروجی را دقیقاً با این ساختار در یک فایل `.json` ذخیره کنید (نمونهٔ
پرشده: `examples/self_compare_example.json`):

```json
{
  "article": {
    "title": "عنوان مقاله",
    "doi": "10.xxxx/xxxxx (اختیاری)",
    "url": "لینک مقاله (اختیاری)"
  },
  "analyst_model": "نام مدل شما (مثلاً claude-sonnet-5)",
  "criteria_rationale": "توضیح آزاد اینکه چرا این معیارها را انتخاب کردید",
  "criteria": [
    {
      "criterion": "نام معیار اول (به زبان خودتان)",
      "raw_score": 6,
      "bsi_score": 8,
      "notes": "توضیح کوتاه چرا این نمرات (اختیاری)"
    }
  ],
  "overall_winner": "bsi",
  "overall_reasoning": "توضیح آزاد قضاوت کلی نهایی"
}
```

## بعد از تولید فایل JSON

```bash
bsi-benchmark self-compare --input my_comparison.json --output results/self_compare/article_1
```

این دستور جدول مقایسه (با ستون «برنده» محاسبه‌شده) را در قالب `.md` و
یک رکورد کامل قابل‌بازخوانی در `.json` تولید می‌کند.
