# Changes — 2026-07-04 session

Summary of what changed in this pass. Follows on from
CHANGES-2026-07-02.md; written so a future session doesn't have to
re-derive this from git log.

## Fixed

- **`compare --bsi-prompt-file` required a manual edit every run**: a raw
  copy of `MASTER_PROMPT_BSI_v3.4.2.md` (or any BSI master prompt) has no
  `{title}`/`{abstract}` placeholders, so `.format()` crashed unless you
  hand-edited the file first. New `prompt_loader.load_bsi_prompt()`
  detects this and auto-appends a placeholder block in memory (never
  touches the file on disk); raises `ValueError` instead of guessing if
  only one of the two placeholders is present (likely a broken hand-edit).
- **arXiv results had no provenance**: `parsers/arxiv.py` never populated
  `Article.doi`/`Article.url`, so there was no way to verify which exact
  paper (or which version of it) a score was computed against. Now
  extracts the canonical `<id>` (abstract-page URL, version-specific) and
  `<arxiv:doi>` when present.
- **`Analysis.generated_at` was defined but never set**: every analysis
  had `generated_at = None` regardless of generator. Now stamped
  centrally in `CrossModelRunner._run_one()` (UTC, ISO 8601) so every
  generator gets a real timestamp without each having to remember to set
  it themselves.

## Added

- **`run_metadata.py`**: `RunMetadata.capture()` snapshots the exact
  `bsi-benchmark` git commit hash (`None` if not a git checkout, not a
  silently-wrong default), whether the working tree was dirty at run
  time, tool version, run timestamp, and a fixed methodology caveat
  (BSIEvaluator is a lexical/keyword proxy, not semantic understanding —
  scores are first-pass triage, not a validated metric, until checked
  against independent human judgement).
- **`--bsi-source-url`** (`compare`): defaults to the canonical BSI repo's
  raw prompt file URL; printed in every report so a reader can go read
  the exact unedited prompt themselves instead of trusting this tool's
  local copy. Pass `''` to omit.
- Both `.md` and `.json` report output now carry: run metadata block,
  BSI prompt source link, and per-article provenance line (doi/url) when
  available — so a report can stand alone as a checkable record of what
  was run, rather than a bare score table.

## Still open / not addressed by this pass

- `BSIEvaluator` itself remains a regex/keyword-based proxy for the seven
  BIO v1.0 dimensions, not a semantic judge. Making numeric BSI scores
  citable as a validated metric (vs. a documented-heuristic triage signal)
  would require a human-judgement validation study, or replacing the
  proxy with an LLM-judge evaluator — both are design decisions, not
  cleanup, and are out of scope here.
- Non-arXiv providers (`crossref` already had doi/url; `openalex`,
  `europepmc`, `semantic_scholar` not audited in this pass for the same
  provenance gap).
- Several `ab/` module files are empty (scaffold-before-content, per
  earlier confirmation from the repo owner — not re-flagged as a defect
  here, listed for visibility only).
